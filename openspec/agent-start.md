# Agent start (OpenSpec)

## Цель
Сделать так, чтобы агент мог:
1) быстро понять, какие фичи уже сделаны (без сканирования всего репо)
2) предложить выбрать фичу для реализации
3) встроить фичу в систему и прогнать quality gates
4) читать релевантные документы во время реализации

## Минимальный сценарий диалога
1. Агент читает `openspec/feature-registry.json`.
2. Агент задаёт один вопрос:
   - **"Какую фичу реализуем? Выберите `featureId` из реестра или скажите название новой фичи."**
3. Если выбрана существующая фича:
   - агент открывает указанные `docs` и релевантные backlog-описания в `D:\Soft-dev\Docs\documents\backlog-descriptions\`.
4. Если это новая фича:
   - агент добавляет запись в `openspec/feature-registry.json` со статусом `planned`,
   - затем создаёт change через OpenSpec.

## Качество
Перед тем как считать change “готовым”:
- агент обязан выполнить шаги из `openspec/quality-gates.md`.
